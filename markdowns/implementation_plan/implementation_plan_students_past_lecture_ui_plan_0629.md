# 학생 VOD(이전 강의) 블록 개편 플랜

현재 수강생의 Home 및 Classroom 탭에 개별 영상이 무작위로 나열되는 구조를 개선하여, 강의 영상이 매일 쌓이더라도 깔끔하게 탐색할 수 있도록 주차별(Week) 그룹화 및 Drill-down UI를 도입합니다.

## User Review Required

> [!IMPORTANT]
> - 주차별 블록을 클릭했을 때 영상 목록을 보여주는 방식에 대해 **"인라인 확장(아래로 펼쳐짐)"** 방식과 **"화면 전환(해당 주차 목록으로 뷰가 바뀜 + 뒤로 가기 버튼)"** 방식 중 **뷰가 바뀌는(Drill-down)** 방식으로 구현하고자 합니다. 모바일 환경에서 더 직관적일 것입니다. 이에 동의하시는지 확인 부탁드립니다.

## Open Questions

> [!WARNING]
> - 상단에 추가되는 **"더 보기"** 버튼을 클릭했을 때는 전체 강의를 리스트 형태로 보여주는 전용 페이지(`/student/lectures` 등)로 이동해야 할까요? 아니면 현재 페이지 내에서 숨겨진 지난 주차들을 더 펼쳐서 보여주는 형태가 나을까요? (일단은 전체 주차를 펼쳐서 보여주는 형태로 제안합니다)

## Proposed Changes

### 1. 데이터 구조 변경 (`types/index.ts`, `data/student.ts`)
- 기존 `RecordedLecture` 및 `ArchivedStream`에 `week` 속성을 추가하여 주차별 데이터를 매핑합니다.
- `data/student.ts`의 `recordedLectures` 모의 데이터를 1주차, 2주차 등 여러 주에 걸쳐 매일매일 업로드된 것처럼 풍부하게 생성합니다.

### 2. 주차별 그룹핑 및 썸네일 자동 갱신 로직
- 컴포넌트 내부에서 전체 영상을 `week` 기준으로 그룹화(Grouping)합니다.
- 각 주차(Week) 블록의 썸네일은 해당 주차 내에서 **가장 최근에 업로드된 영상(`date` 기준 최신순 정렬)의 썸네일**로 자동 지정되도록 로직을 구현합니다.

### 3. UI 컴포넌트 업데이트 (`Home.tsx`, `Classroom.tsx`)
- **타이틀 영역**: 우측 상단에 "더 보기" 버튼/아이콘을 추가합니다.
- **기본 뷰 (Week Grid 뷰)**: 개별 영상 대신 `[1주차: React 기초]`, `[2주차: 상태 관리]` 등의 주차별 썸네일 카드 묶음을 먼저 보여줍니다.
- **상세 뷰 (Lecture List 뷰)**: 주차 블록을 클릭하면 해당 주에 포함된 개별 영상들이 쭈루룩 나열되는 리스트 뷰로 전환되며, 상단에 `[뒤로 가기]` 버튼을 두어 다시 주차별 뷰로 돌아올 수 있게 합니다.

### Manual Verification

- VOD 영역에 "주차별" 썸네일이 가장 최신 영상의 썸네일로 잘 반영되어 있는지 확인.
- 주차 카드를 클릭했을 때 해당 주차의 영상 리스트로 화면이 부드럽게 전환되는지 확인.

# 🖥️ Ollama Local LLM 하드웨어 맞춤형 모델 추천 보고서

본 보고서는 작성하신 두 가지 하드웨어 환경(회사 및 자택)의 스펙 차이를 고려하여, 향후 파이프라인 연동 및 고도화 테스트에 가장 적합한 **Ollama 기반 로컬 LLM 모델**들을 제안합니다.

---

## 1. 하드웨어 환경 분석

| 환경 | VRAM | 시스템 RAM | 목적 및 특징 |
|:---|:---|:---|:---|
| **회사 PC** | **8GB** | 16GB | **파이프라인 통신 검증 및 기초 테스트용**. VRAM이 제한적이므로 무거운 모델 구동 시 Out Of Memory(OOM)가 발생하거나 급격한 속도 저하가 올 수 있습니다. **10B(100억 개) 파라미터 미만의 4-bit 양자화(Quantization) 모델** 사용이 필수적입니다. |
| **자택 PC** | **16GB** | **64GB** | **응답 품질 고도화 및 복잡한 추론용**. VRAM 16GB로 14B~32B 모델의 일부를 커버할 수 있으며, 방대한 64GB 시스템 RAM을 활용해(일부 VRAM 오프로딩) 더 큰 모델(예: 70B, Mixtral)도 약간의 속도 타협과 함께 충분히 구동 가능합니다. |

> [!IMPORTANT]
> Ollama는 모델을 구동할 때 VRAM을 최우선으로 사용하며, VRAM이 꽉 차면 남은 모델 레이어를 시스템 RAM으로 넘깁니다(Offloading). RAM으로 넘어가면 속도는 느려지지만 모델은 멈추지 않고 구동됩니다. 하지만 회사 PC는 RAM도 16GB이므로 철저하게 **가벼운 모델**로 파이프라인의 입출력이 정상적으로 돌아가는지 확인하는 데 집중해야 합니다.

## 2. 🏢 회사 PC (VRAM 8GB) 추천 모델: "파이프라인 검증용"

이 단계에서는 모델의 지능(답변의 유려함)보다는 **빠른 속도, 낮은 리소스 점유, JSON 출력 및 함수 호출(Tool Use) 지원 여부**가 중요합니다.

### ① Qwen 2.5 (7B) 🌟 (적극 추천)
* **특징**: 알리바바에서 만든 최신 모델로, 7B(70억) 파라미터급에서 **가장 압도적인 한국어 성능과 코딩/JSON 구조화 능력**을 보여줍니다.
* **리소스**: 약 4.7GB VRAM 점유 (8GB 환경에서 매우 쾌적)
* **실행 명령어**: `ollama run qwen2.5:7b`

### ② Llama 3.1 (8B)
* **특징**: 메타(Meta)의 오픈소스 LLM 생태계 표준입니다. Tool Use(함수 호출) 능력이 매우 뛰어나 파이프라인 연동 시 안정적인 프롬프팅이 가능합니다. 단, 한국어 능력이 Qwen2.5에 비해서는 약간 아쉬울 수 있습니다.
* **리소스**: 약 4.7GB VRAM 점유
* **실행 명령어**: `ollama run llama3.1`

### ③ DeepSeek-R1-Distill (1.5B / 7B / 8B)
* **특징**: 최근 전 세계적으로 돌풍을 일으키고 있는 DeepSeek-R1의 추론(Reasoning) 과정을 Qwen이나 Llama에 증류(Distill)한 모델입니다. 작은 체급에서도 깊은 논리적 사고 과정을 거친 후 답변을 내놓습니다.
* **리소스**: 1.5B(~1GB), 7B/8B(~4.7GB)
* **실행 명령어**: `ollama run deepseek-r1:7b` 또는 `ollama run deepseek-r1:8b`

### ④ Phi-3.5 Mini (3.8B)
* **특징**: 마이크로소프트의 초경량 모델입니다. VRAM을 고작 2.3GB 정도만 차지하므로 서버와 다른 작업을 동시에 띄워놓고 프론트/백엔드 코딩을 할 때 PC에 전혀 무리를 주지 않습니다. **단순 API 통신 성공 여부**만 테스트할 때 최고입니다.
* **실행 명령어**: `ollama run phi3.5`

## 3. 🏠 자택 PC (VRAM 16GB / RAM 64GB) 추천 모델: "품질 고도화용"

자택 PC는 VRAM 16GB와 넉넉한 64GB RAM을 바탕으로, 실제 서비스에 배포할 만한 고지능 모델을 돌려보며 프롬프트를 튜닝하고 RAG(검색 증강 생성) 품질을 높이는 데 사용합니다.

### ① Qwen 2.5 (14B / 32B)
* **특징**: 14B 모델은 VRAM 16GB에 거의 딱 맞게 들어가며 속도와 성능 밸런스가 완벽합니다. 32B 모델은 VRAM을 약간 초과하여 시스템 RAM(64GB 중 일부)을 끌어 쓰게 되지만, 답변 퀄리티가 GPT-4 초창기 버전에 필적할 만큼 매우 뛰어납니다.
* **실행 명령어**: `ollama run qwen2.5:14b` 또는 `ollama run qwen2.5:32b`

### ② DeepSeek-R1-Distill (14B / 32B)
* **특징**: 복잡한 비즈니스 로직이나 수학적, 논리적 판단이 필요한 에이전트 파이프라인을 구축할 때 가장 뛰어난 성능을 발휘합니다.
* **실행 명령어**: `ollama run deepseek-r1:14b` 또는 `ollama run deepseek-r1:32b`

### ③ Command-R (35B)
* **특징**: Cohere사에서 만든 **RAG(검색 증강 생성) 및 Tool Use 특화 모델**입니다. 회사 문서를 검색해서 답변하는 시스템(RAG)을 만들 계획이시라면 이 모델의 품질이 압도적입니다. 시스템 RAM을 활용하여 구동 가능합니다.
* **실행 명령어**: `ollama run command-r`

## 4. 🚀 개발 파이프라인 전략 제언

1. **로컬 개발 (회사)**: 백엔드(.env)의 LLM 모델 설정을 `qwen2.5:7b` 또는 `phi3.5`로 지정합니다. 프롬프트가 정상적으로 서버에 전달되고, LLM이 응답을 반환하여 프론트엔드 UI에 뿌려지는 **통신 파이프라인과 파싱 로직**만 검증합니다.
2. **품질 튜닝 (자택)**: 동일한 코드 베이스에서 모델 설정만 `qwen2.5:32b` 또는 `deepseek-r1:14b`로 변경합니다. 프롬프트 엔지니어링을 정교하게 다듬어, "기대하는 형태의 고품질 한국어 응답"이 나오도록 튜닝합니다.

위 내용을 바탕으로 이후 백엔드와 Ollama를 연결하는 LangChain 또는 기본 HTTP 파이프라인 설계 로직을 반영해 드릴 수 있습니다. 확인해 보시고 추후 파이프라인 설계가 필요하실 때 언제든 말씀해 주세요!

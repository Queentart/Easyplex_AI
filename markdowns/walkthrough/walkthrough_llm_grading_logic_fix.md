# AI 멀티모달 파일 분석 시스템 연동 워크스루

제안해 주신 멀티-파일(Multi-file) 분석 채점 로직 구현이 성공적으로 완료되었습니다! 🎉
이제 AI 자동 채점(Auto-Grade) 버튼을 누르면, 백엔드 서버가 학생이 업로드한 **모든 유형의 파일 포맷을 실시간으로 감지**하여 각 파일에 알맞은 파이프라인으로 내용을 추출해 낸 뒤 LLM에게 평가를 지시합니다.

## 🛠 주요 구현 내용 요약

### 1. 전용 파일 파서 엔진 도입 (`file_parser.py`)
백엔드 코어에 `parse_file_for_llm`이라는 전용 유틸리티 함수를 새로 설계했습니다. 업로드된 파일의 타입별로 처리 로직이 분기됩니다.
- **문서 및 코드 (`.py`, `.js`, `.txt` 등)**: 파일 내의 실제 텍스트 데이터를 그대로 읽어내어 LLM 컨텍스트에 삽입합니다.
- **PDF 문서 (`.pdf`)**: `pypdf` 라이브러리를 활용하여 PDF 내의 글씨들을 텍스트로 추출(Extraction)합니다. (과부하 방지를 위해 최대 10페이지로 제한)
- **오디오 파일 (`.mp3`, `.wav` 등)**: `SpeechRecognition` 및 `pydub`를 통해 오디오 파일을 WAV로 변환 후, STT(음성 인식)를 돌려 오디오 속 대화 내용을 스크립트로 번역하여 LLM에게 제공합니다.
- **이미지 및 비디오**: 파일의 해상도, 크기, 포맷 등 상세 메타데이터를 추출하여 AI에게 전달합니다. (완전한 비디오/이미지 이해를 위해서는 강사님의 수동 평가와 결합할 수 있도록 안내 문구를 추가했습니다.)

### 2. 채점 프롬프트 확장 연동 (`assignments.py`)
기존에 `generate-grading` API는 텍스트만 처리했지만, 이제 학생의 `file_url`을 받아 서버의 로컬 저장소에서 해당 파일을 꺼내옵니다. 파서 엔진을 거쳐 텍스트화된 결과물(`parsed_text`)은 아래와 같은 프롬프트 형태로 LLM(Ollama)에게 함께 전달됩니다.

```text
Student: 김지우
Assignment: 과제1 테스트
Text Submission: (학생이 입력한 메모 텍스트)

Attached File (hello_world.py):
[Text/Code File Content]
print("Hello, World!")
```

### 3. 프론트엔드 연동 완료
- `AssignmentsGrading.tsx`와 `assignmentApi.ts`에서 채점 시 `file_url`과 `file_name` 매개변수를 백엔드로 안전하게 던지도록 API 구조가 연동되었습니다.

> [!TIP]
> 이제 코딩 과제, PDF 리포트 파일 등을 올리더라도 **강사님이 파일을 직접 열어보시기 전에 AI가 먼저 파일을 읽고 점수와 피드백을 예측**해 냅니다! 직접 PDF 파일이나 간단한 Python 파일을 학생 계정으로 업로드한 뒤, 강사 계정에서 Auto-Grade를 클릭하여 파일 내용을 잘 인식하는지 테스트해 보세요.

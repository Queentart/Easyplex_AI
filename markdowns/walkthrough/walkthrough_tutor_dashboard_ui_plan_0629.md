# 보조강사/멘토(Tutor) 맞춤형 대시보드 구현 완료 안내

요청하신 **보조강사/멘토(Tutor) 맞춤형 통합 대시보드** 뷰 분리 및 UI 구현을 모두 완료했습니다.

## 핵심 변경 내용

### 1. 역할 기반(Role-based) 대시보드 뷰 분리
- **`InstructorDashboard.tsx`**: 이제 로그인한 유저의 역할(`role`)을 확인하여, 주강사(`instructor`)일 경우 `<InstructorDashboardView />`를 렌더링하고, 멘토(`tutor`)일 경우 새롭게 생성한 `<TutorDashboardView />`를 렌더링하도록 동적으로 분기 처리했습니다.
- 헤더 문구 역시 멘토 로그인 시 **"EduAI Tutor", "Mentoring Dashboard"** 등으로 적절하게 노출되게 수정했습니다.

### 2. 멘토 맞춤형 UI 구조 재배치 (`TutorDashboardView.tsx`)
보조강사(Tutor)의 핵심 업무 흐름인 학생 케어와 일지 관리에 최적화된 레이아웃을 구성했습니다.

- **High Priority (최상단, 강조 카드)**
  - **멘토링 일지 현황**: 가장 중요한 업무인 일지 제출 현황을 상단에 크게 배치했습니다. 마감이 임박한 일지는 붉은색(`Urgent`)으로 표시되며, 초록색 보더(Border) 포인트를 주었습니다.
  - **진행 예정 멘토링 (학생 상담)**: 본인에게 배정된 학생과의 상담 일정을 확인하고, 이슈 및 AI 추천 가이드를 한눈에 파악할 수 있도록 붉은색 보더 강조 처리를 했습니다.
  - **학습 질문 게시판**: 답변 대기 중인 질문을 빠르게 파악할 수 있도록 노란색 보더 포인트와 함께 상단에 배치했습니다.

- **Medium Priority (중단 영역)**
  - **과제 보조 채점**: 본인에게 할당된(Assigned) 1차 피드백 혹은 채점 대기 건수와 전체 채점 마감 현황을 배치했습니다.
  - **최근 업로드된 강의 자료**: 수업을 보조하기 위해 필요한 최신 자료 목록을 제공합니다.

- **Low Priority (하단 영역)**
  - **Quick Links**: 실시간 스트리밍 접속과 학습 커리큘럼 로드맵 확인용 버튼을 아기자기하게 하단에 배치했습니다.

### 3. 모의 데이터 및 스타일링 업데이트
- 멘토 전용 일지 알림과 진행 예정 세션 목록을 표현하기 위해 `instructor.ts` 내에 `tutorMentoringSessions` 및 `tutorPendingLogs` 더미 데이터를 추가했습니다.
- 뷰가 분리되면서 강조용 CSS 클래스 충돌이 없도록, `nth-child` 대신 `.card-highlight-success`, `.card-highlight-danger`, `.card-highlight-warning` 등의 명시적인 클래스로 리팩토링하여 레이아웃의 안정성을 높였습니다.

## 테스트 방법
1. 로컬 개발 서버(`npm run dev`)를 실행합니다.
2. 현재 `InstructorDashboard.tsx`의 12번째 줄 근처의 `const isTutor = currentUser.role === 'tutor';` 조건을 임의로 `const isTutor = true;` 로 변경하시거나, 멘토 권한의 더미 유저 데이터를 넣어 `/instructor`에 진입해 보세요.
3. 멘토 맞춤형 레이아웃과 데이터가 제대로 렌더링되고 강조 색상이 의도대로 나오는지 확인해 주시면 됩니다.

> [!TIP]
> 주강사 뷰와 동일하게 애니메이션 및 호버 효과가 적용되어 있습니다. 컴포넌트를 깔끔하게 분리해 두었으므로, 나중에 각 뷰별로 기능이 추가/수정되더라도 서로 충돌 없이 유지보수가 가능합니다!

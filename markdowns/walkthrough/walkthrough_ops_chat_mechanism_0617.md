# Operations Team Inquiry Workflow Walkthrough

수강생과 운영팀 간의 문의(Ticket) 처리 워크플로우 개발이 모두 성공적으로 완료되었습니다! 
프론트엔드부터 백엔드, 데이터베이스까지 아래의 내용들이 어떻게 적용되었는지 안내해 드립니다.

## 1. 백엔드 및 데이터베이스 (Backend & DB)
- **OpsTicket 데이터 모델:** `backend/app/models/ops.py`에 수강생의 문의 내역과 운영팀의 답변 내역, 처리 상태(`pending`, `answered`)를 저장할 수 있는 PostgreSQL DB 모델을 새롭게 정의했습니다.
- **수강생 API 업데이트 (`/support/message` & `/tickets`):** 
  - 수강생이 채팅창에서 **Operations Team** 채널을 선택하고 메시지를 보내면, DB에 새로운 티켓(Ticket) 데이터가 생성되도록 로직을 추가했습니다.
  - 챗봇이 주기적으로(5초마다) 나의 문의 내역과 운영팀의 답변 여부를 체크해 올 수 있도록 조회 엔드포인트를 추가했습니다.
- **운영팀 전용 API 구축 (`/ops/inquiries`):** 
  - 운영팀 대시보드에서 모든 학생의 문의 내역을 시간순으로 불러오고, 특정 티켓에 답변을 달 수 있는 RESTful API를 새롭게 구현했습니다.

> [!CAUTION]
> 데이터베이스 모델이 새로 추가되었기 때문에 테스트를 위해선 앞서 FAQ 모델 추가 때 하셨던 것처럼 마이그레이션(Alembic)을 한 번 더 실행해 주셔야 합니다! (하단 테스트 방법 참고)

---

## 2. 운영팀(EduOps) 대시보드 UI
- **사이드바 메뉴 업데이트:** 운영팀이 좌측 사이드바에서 **[문의사항 (Inquiries)]** 메뉴를 클릭하여 바로 접속할 수 있게 되었습니다.
- **Inquiries.tsx 신규 화면 추가:** 
  - 새롭게 접수된 학생들의 문의(대기 중)와 이미 답변이 완료된 문의를 시각적으로 구별하여 목록 형태로 보여주는 페이지를 제작했습니다.
  - 리스트 내의 **[답변 작성하기]** 버튼을 누르면 입력란이 나타나고, 답변을 등록하면 상태가 초록색 '답변 완료' 뱃지로 실시간 변경됩니다!

---

## 3. 수강생 채팅 모달 (AIHelpBot.tsx)
- 학생이 채팅 모달을 띄워두고 있을 때, 뒤에서 **조용히(Polling)** 자신의 티켓 상태를 가져옵니다.
- 만약 내가 남긴 문의사항에 운영팀이 답변을 달아 주었다면, 챗봇 대화창 하단에 즉시 **`[운영팀 답변 도착 알림 🔔]`** 이라는 이름으로 시스템 메시지 버블이 생성됩니다!

---

## 🛠️ How to Test (테스트 가이드)

테스트하시기 전 반드시 터미널에서 **DB 마이그레이션 명령어**를 실행해 주세요!

```bash
# 가상환경 활성화 후 (Windows PowerShell 기준)
..\venv\Scripts\python -m alembic revision --autogenerate -m "Add OpsTicket model"
..\venv\Scripts\python -m alembic upgrade head
```

1. **수강생 시점:** 학생 페이지를 열고 우측 하단 챗봇 버튼 클릭 ➡️ 상단 가로 스크롤에서 **Operations Team** 채널 선택 ➡️ 아무 문의나 입력 후 전송합니다.
2. **운영팀 시점:** 다른 탭에 운영팀 관리자 주소(`http://localhost:3000/eduops/inquiries` 또는 `/eduops` 접근 후 좌측 사이드바 클릭)로 접속합니다.
3. 방금 학생이 남긴 질문을 확인하고 **답변 작성하기** 버튼을 눌러 답변을 등록합니다.
4. 다시 **수강생 시점** 화면으로 돌아가 보면, 잠시 뒤 챗봇 화면 맨 아래에 운영팀의 답변이 알림 메시지로 자동으로 쓱 나타나는 것을 확인하실 수 있습니다!

# TechOps 스트리밍 & 수강생 VOD 연동 개편 완료 안내

기술팀(TechOps)의 스트리밍 관리 권한이 수강생의 "지난 강의 영상(VOD)" 메뉴와 유기적으로 연동될 수 있도록 모든 기능 구현을 완료했습니다.

## 핵심 적용 사항

### 1. 송출 종료(Archive) 시 VOD 자동 아카이빙 연동
- 기술팀이 `Streaming Management` 페이지에서 라이브 방송을 진행한 후 **[송출 종료 처리 (Archive)]** 버튼을 누르면, 해당 방송의 정보(타이틀, 날짜, 유튜브/비메오 링크 등)가 자동으로 시스템의 VOD 목록으로 이관됩니다.
- 이때 주차(Week)는 기존에 존재하는 VOD들의 가장 높은 주차(가장 최근 주차)로 자동 배정됩니다.

### 2. VOD(과거 강의) 관리자 패널 신설
- 기술팀 대시보드 하단에 **[VOD / Archived Lectures Management]** 테이블 패널을 새로 추가했습니다.
- 기술팀은 여기서 기존에 올라간 VOD의 주차, 타이틀, 링크들을 클릭 한 번(`Edit`)으로 쉽게 수정할 수 있으며, 누락된 이전 강의를 수동으로 업로드할 수 있습니다.
- **실시간 동기화:** 여기서 수정한 모든 내역은 브라우저 저장소 동기화(`vodService`)를 통해 수강생의 화면(Home 탭, Classroom 탭)을 즉시 리렌더링하여 실시간으로 반영됩니다.

### 3. YouTube / Vimeo 링크 폴백(Fallback) 메커니즘
- 수강생 뷰(`WeeklyVODBlock`)의 개별 영상 링크 클릭 시, 시스템이 내부적으로 우선순위를 판단합니다.
  - 최우선: **YouTube Live URL**
  - 차선책 (유튜브가 없거나 끊긴 경우 대체): **Vimeo Live URL**
- 이제 기술팀이 VOD 설정에서 유튜브 링크를 삭제하고 비메오 링크만 넣어두면, 수강생이 버튼을 눌렀을 때 자동으로 비메오로 연결됩니다.

## 동작 테스트 권장
1. 기술팀 뷰(`/techops/streams`) 접속
2. 현재 송출 중인 임의의 라이브 방송을 켠 뒤, **[송출 종료 처리]**를 클릭해보세요.
3. 알림창과 함께 하단 VOD 리스트 맨 위에 새 영상이 추가되는 것을 볼 수 있습니다.
4. 수강생 뷰(`/student`)를 새 탭으로 열어보시면 새로고침을 하지 않아도 VOD 블록이 해당 영상으로 업데이트된 것을 확인하실 수 있습니다.

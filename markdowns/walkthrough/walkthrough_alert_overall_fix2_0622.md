# Notification UI & Structure Updates Walkthrough

승인해 주신 플랜에 따라 알림 시스템의 UI를 오버레이(드롭다운) 형태로 개편하고, 5가지 알림 유형에 대한 구조적 확립을 모두 완료했습니다.

## 1. UI 개편: 알림 드롭다운 (Notification Dropdown)
기존에는 우측 상단의 🔔(종 모양) 아이콘 클릭 시 `/notifications`라는 별도의 전체 화면 페이지로 이동했지만, 이제는 **현재 보고 있는 화면을 방해하지 않는 깔끔한 팝업 드롭다운**이 나타나도록 수정했습니다.

- 뒷 배경이 딤(Blur) 처리되지 않아 학습 자료나 채팅을 보면서도 동시에 알림을 확인할 수 있습니다.
- 팝업 바깥쪽을 클릭하면 자연스럽게 닫히도록(Click Outside) UX를 개선했습니다.
- 최신 알림 목록과 스크롤, `테스트` 버튼, `모두 읽음` 기능이 작은 팝업 공간 안에 완벽하게 최적화되어 이식되었습니다.

## 2. 5가지 알림 유형 매핑 및 순환 테스트
제안해 주신 목적에 맞게 시스템 내부적으로 알림 유형(`info`, `success`, `alert`, `message`, `warning`)을 명확히 정의했습니다.

- **`warning` 알림 제안 채택**: 제안해 드렸던 `과제 마감 기한 임박 알림`, `시스템 점검 예정 알림` 등의 목적으로 활용할 수 있도록 확립했습니다.
- **[테스트] 버튼 로직 개선**: 기존 랜덤 생성 방식에서 벗어나, 버튼을 누를 때마다 **5가지 유형이 순서대로 하나씩 생성**되도록 개선했습니다. (이를 통해 파란색 말풍선 아이콘의 `message` 알림도 즉시 확인 가능합니다.)

## 3. Global 알림 발송 아키텍처 추가
백엔드 단에서 공지사항 등록 등 전체 수강생을 대상으로 한꺼번에 알림을 보낼 때 활용할 수 있도록 `create_global_notification` 헬퍼 함수를 새롭게 설계하여 `notification_service.py`에 배포했습니다. 
향후 운영팀 대시보드 등에서 전체 공지를 띄울 때 이 함수가 호출되어 전체 브로드캐스트가 처리됩니다.
